# cvService

- Веса YOLO: **`models/best.onnx`** (каталог `./cvService/models` → в контейнере `/app/models`). Ожидается экспорт в духе **Ultralytics YOLO**: вход **`images`** `[1,3,H,W]`, выход **`output0`** `[1, 4+nc, N]` (например `nc=2` → `[1,6,8400]`); классы — **логиты**, в коде применяется **sigmoid**. Имена классов в `ONNXYOLODetector.class_names` должны совпадать с порядком `nc` при обучении.
- **Нет файла весов:** по умолчанию используется **заглушка** (нулевые метрики, HTTP 200), цепочка vineyardApp → MinIO → CV не падает. Явно требовать ошибку без весов: **`CV_STRICT_MODEL=1`**.
- **`CV_DUMMY_INFERENCE`** необязателен (оставлен для совместимости и логов). После появления `best.onnx` загрузится реальный ONNX; контейнер перезапустите, чтобы подхватить новый код/файлы.
