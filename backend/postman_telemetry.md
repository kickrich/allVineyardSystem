# Тестирование API Телеметрии в Postman

Этот документ описывает, как протестировать прием и хранение телеметрии дронов через API.

## 1. Отправка данных телеметрии (POST /telemetries)


Используйте этот запрос, чтобы имитировать дрон, отправляющий свои текущие данные.

**Метод:** `POST`
**URL:** `http://localhost:3000/api/v1/telemetries`


**Body (JSON):**
```json
{
  "telemetry": {
    "mission_id": 1,
    "recorded_at": "2026-02-25T12:34:56Z",
    "latitude": 59.8123,
    "longitude": 30.3456,
    "altitude": 75.5,
    "speed": 12.3,
    "battery": 68
  }
}
```

**Ожидаемый результат:**
*   **Status Code:** `201 Created`
*   **Response Body:** Должен содержать объект `telemetry` с переданными данными и присвоенным `id`.


> **Важно:** Замените `mission_id` на реальный ID миссии, которая в данный момент выполняется. Поле `recorded_at` может быть опущено, в этом случае будет использовано текущее время сервера.


---


## 2. Получение списка всей телеметрии (GET /telemetries)


Этот запрос позволяет получить все записи телеметрии.

**Метод:** `GET`
**URL:** `http://localhost:3000/api/v1/telemetries`


**Ожидаемый результат:**
*   **Status Code:** `200 OK`
*   **Response Body:** Массив всех объектов `telemetry`.


---


## 3. Получение конкретной записи телеметрии (GET /telemetries/:id)


Используйте этот запрос для получения данных одной конкретной телеметрии.

**Метод:** `GET`
**URL:** `http://localhost:3000/api/v1/telemetries/1` *(замените `1` на нужный ID)*


**Ожидаемый результат:**
*   **Status Code:** `200 OK`
*   **Response Body:** Один объект `telemetry` с указанным ID.


---


## 4. Обновление записи телеметрии (PATCH /telemetries/:id)


Хотя в реальности телеметрию обычно не обновляют, API позволяет это сделать.

**Метод:** `PATCH`
**URL:** `http://localhost:3000/api/v1/telemetries/1` *(замените `1` на нужный ID)*


**Body (JSON):**
```json
{
  "telemetry": {
    "speed": 15.0
  }
}
```

**Ожидаемый результат:**
*   **Status Code:** `200 OK`
*   **Response Body:** Объект `telemetry` с обновленным значением.


---


> **Примечание:**
> 
> Логика хранения истории перемещения реализована автоматически: каждая новая запись телеметрии сохраняется в базе данных, создавая историю. Чтобы построить трек (путь дрона), фронтенд должен запросить все записи телеметрии для конкретной миссии и отсортировать их по полю `recorded_at`.
