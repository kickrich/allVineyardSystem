# Задачи для Backend (Ruby on Rails)

## 1.4.3 Привязка к зоне

### Контекст

У нас есть конкретный, фиксированный участок виноградника, где разрешено выполнять миссии дронов. Это не динамическая зона для каждой миссии, а одна основная зона "работы".

### Решение

Нет необходимости создавать отдельную модель `Zone` или добавлять поле `polygon` в `Mission`. Это было бы избыточно, так как зона у нас одна и та же для всех миссий.


Вместо этого, нужно:

1.  **Определить координаты полигона виноградника.** Это можно сделать вручную с помощью Яндекс.Карт (или другого инструмента) и захардкодить их в коде.
2.  **Создать валидацию в модели `Route`.** Эта валидация будет проверять, что точка маршрута находится внутри этого фиксированного полигона.

3.  **Адаптировать интеграцию с Yandex Maps API.** Сервис `YandexMapsService` нужен не для получения полигона зоны (он у нас фиксированный), а для получения **максимальной допустимой высоты (`max_altitude`)** для заданной координаты. Это позволит дрону летать безопасно, учитывая рельеф и постройки.


### Конкретные действия

1.  **Найти границы участка виноградника на Яндекс.Картах.**
    *   Найди на карте участок виноградника.
    *   Определи несколько контрольных точек (минимум 4), которые образуют полигон, охватывающий весь участок.
    *   Запиши их координаты (широту и долготу).

2.  **Обновить валидацию в модели `Route`.**
    *   Удали старую валидацию (если она была), которая проверяла высоту только по жестким пределам.
    *   Добавь новую валидацию, которая:
        *   Проверяет, находится ли точка `latitude`, `longitude` внутри фиксированного полигона виноградника.
        *   Проверяет, что `altitude` не превышает `max_altitude`.

3.  **Реализовать сервис `YandexMapsService`.**
    *   Создай файл `app/services/yandex_maps_service.rb`.
    *   Напиши метод, например `get_max_altitude(latitude, longitude)`, который будет вызывать API Яндекс.Карт (например, [Geocoder API](https://yandex.ru/dev/maps/geocoder/)) для получения высоты над уровнем моря в заданной точке, и добавлять к ней безопасный зазор (например, 50 метров).
    *   Этот метод будет вызываться при валидации точки маршрута.

4.  **Обновить контроллер `routes_controller.rb`.**
    *   При создании `Route`, перед сохранением, нужно вызвать `YandexMapsService` для получения `max_altitude` и установить это значение в модель. Затем валидация сама проверит, что `altitude` пользователя <= `max_altitude`.

---

> **Вывод:** Создавать модель `Zone` не нужно. Работаем с одной фиксированной зоной и используем API карт для получения данных о высоте, а не границах.
